## Description

Design a logged-out home page (the `/` route for unauthenticated visitors) that explains what Median is and why someone would want to use it. This is the page a new user sees before they sign up — including someone who received a list invite and followed a link.

## Context

The initial user base is small and known (invited users), but even invited users need to understand what they're signing up for before creating an account. A blank or purely functional landing page misses the opportunity to set expectations and build confidence.

The tone of the app is playful and irreverent — the landing page should reflect that rather than being generic SaaS copy.

## Questions to resolve

- What is the core value prop in one sentence? (Decision support + shared memory for what to watch — but how do you say that in a way that's compelling?)
- What does the page actually show? (Hero + CTA, feature highlights, screenshot/mockup, all three?)
- Does the page handle the "I was invited" case specially? (e.g. "X invited you to a list on Median" as the hero message when arriving from an invite link)
- What are the CTAs? (Sign up, Log in — or just Sign up with Log in as a secondary link?)
- Does the page need to explain the enthusiasm score / collaborative list concept, or is "sign up and see" enough for an invited user?
